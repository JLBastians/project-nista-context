# Backend Changelog — 2026-04-11

## [Shopify Storefront API Integration] — Stages 1-4 Implementation

**Request:** Begin extracting product data from Shopify tokenless Storefront API. Establish a complete, consistent data pipeline from Shopify → PostgreSQL → FastAPI → Next.js frontend. Critical requirement: schema field naming must be consistent across all three systems.

**Solution:** Implemented all four stages of the Shopify integration plan in one pass. All new Shopify-sourced columns use a `shopify_` prefix for clear data provenance. The Pydantic layer maps prefixed DB columns to clean frontend-friendly names. All existing scraper code (Myer, Revolve) is untouched.

---

### Stage 1 — Schema & Contracts

**[models.py] — Add Brand model**
- New `Brand` SQLAlchemy model with table `brands` (brand_id, name, myshopify_domain, storefront_api_version, is_active, fetch_interval_hours, created_at, updated_at)
- One row per Shopify store; FK referenced by products.brand_id

**[models.py] — Extend Product model with 15 new columns**
- 10 Shopify-sourced columns (all prefixed `shopify_`): `shopify_gid`, `shopify_handle`, `shopify_title`, `shopify_product_type`, `shopify_description`, `shopify_currency`, `shopify_compare_at_price`, `shopify_variants` (JSONB), `shopify_created_at`, `shopify_updated_at`
- 5 STROBE-internal columns: `brand_id` (FK), `subcategory`, `fetched_at`, `expires_at`, `enrichment_status`
- Added `__table_args__` with `UniqueConstraint` on `shopify_gid` and `CheckConstraint` on `enrichment_status`

**[migrate_shopify_schema.py] — New migration script**
- Created `Backend/scripts/migrate_shopify_schema.py`
- Follows existing pattern from `migrate_add_embedding_metadata.py`
- 22 migration steps: CREATE TABLE brands, 15× ALTER TABLE products ADD COLUMN, 2× ADD CONSTRAINT, 4× CREATE INDEX
- Supports `--dry-run` flag; safe to re-run (checks existence before each step)
- **Run this first:** `cd Backend && python scripts/migrate_shopify_schema.py`

**[main.py] — Updated Pydantic schemas**
- `ProductCreate`: extended with all 15 new column fields (all optional, backward compatible)
- `ProductResponse`: added `model_validator(mode='before')` that maps `shopify_*` DB column names to clean frontend names: `shopify_title→name`, `shopify_description→description`, `shopify_currency→currency`, `shopify_compare_at_price→compare_at_price`, `shopify_handle→handle`, `shopify_variants→variants`; `shopify_gid` kept as-is (needed for cart API)
- Added imports: `model_validator`, `Any`

**[crud.py] — Extended create_product()**
- Added 15 new keyword arguments matching the new columns
- Passes all new fields to `models.Product(...)` constructor

**[types.ts] — Added ShopifyVariant interface**
- New `ShopifyVariant` TypeScript interface in `Frontend/src/lib/types.ts`
- Fields: `shopify_variant_gid`, `title`, `available`, `price`, `compare_at_price`, `sku`, `selected_options`
- Used internally for size/colour selection and Shopify cart API line item creation

### Stage 2 — Shopify GraphQL Client

**[shopify_client.py] — New Shopify API client**
- Created `Backend/scrapers/shopify_client.py`
- `ShopifyClient` class with `__init__`, `fetch_page`, `fetch_all_products` (generator), `_handle_errors`
- No auth headers — tokenless access only (Content-Type: application/json)
- 5-second delay between pages (`min_delay=5`), matching existing scraper convention
- HTTP 429: 30-second back-off, up to 3 retries per page
- Custom exceptions: `ShopifyClientError`, `ShopifyRateLimitError`, `ShopifyGraphQLError`
- GraphQL query fetches: id, handle, title, description, vendor, productType, availableForSale, onlineStoreUrl, createdAt, updatedAt, priceRange, compareAtPriceRange, images(first:10), variants(first:50) with selectedOptions

**[shopify_transform.py] — New data transformer**
- Created `Backend/scrapers/shopify_transform.py`
- Pure functions, no DB access — fully testable in isolation
- `transform_product(node, store_config)` → DB column dict with all `shopify_*` fields
- `extract_variants(edges)` → flat JSONB list with shopify_variant_gid, title, available, price, compare_at_price, sku, selected_options
- `map_category(product_type)` → (category, subcategory) using a 70+ entry lookup dict
- `compute_expires_at(fetched_at, hours)` → fetched_at + N hours
- `structured_tags` populated with Shopify provenance metadata for enrichment pipeline compatibility

### Stage 3 — Ingestion Pipeline

**[db_helpers.py] — Added upsert_shopify_products()**
- New function added alongside existing helpers (existing functions untouched)
- Deduplicates by `shopify_gid` column (NOT `structured_tags` JSONB — that stays for Myer/Revolve)
- INSERT if GID not found → `enrichment_status='pending'`
- UPDATE if GID found AND (expires_at passed OR shopify_updated_at changed)
  - Resets enrichment_status to 'pending' ONLY if price or primary image changed
- SKIP if snapshot is fresh and product unchanged
- Returns `{inserted, updated, skipped, errors}` dict

**[store_registry.py] — New brand registry**
- Created `Backend/scrapers/store_registry.py`
- `INITIAL_BRANDS` list (Gymshark, Allbirds, Reformation — all marked `is_active=False`)
- Brands default to inactive; set `is_active=True` only after verifying tokenless access with the curl command documented in the file header
- `seed_brands(db)` — inserts INITIAL_BRANDS, skips existing
- `get_active_brands(db)` — returns active brands as config dicts
- `get_brand_by_name(db, name)` — fetches a single brand by name

**[shopify_ingest.py] — New main ingestion CLI**
- Created `Backend/scrapers/shopify_ingest.py`
- CLI flags: `--brand NAME`, `--all`, `--seed-brands`, `--limit N`, `--dry-run`, `--save-json`, `--force-refresh`
- Ingestion loop: fetch pages → transform → batch upsert every 100 products
- 5-second delay between brands; graceful per-brand error handling (continues to next brand on failure)
- JSON backup via existing `db_helpers.save_to_json()` when `--save-json` is set
- Console output matches existing scraper style (header/progress/summary blocks)

### Stage 4 — E2E Validation

**[test_shopify_e2e.py] — New diagnostic script**
- Created `Backend/tests/test_shopify_e2e.py`
- 6 test groups: API connectivity, transform correctness, DB round-trip, API response shape, enrichment pipeline compatibility, dedup/freshness
- Uses a hardcoded `FIXTURE_NODE` for offline transform tests (no live API required for Tests 2-6)
- Test 4 (API response shape) requires the FastAPI server running on localhost:8000; gracefully skipped if not reachable
- Cleans up E2E test products after each run (identified by `E2E_TEST_` GID prefix)
- Exits with code 0 if all checks pass, 1 if any fail
