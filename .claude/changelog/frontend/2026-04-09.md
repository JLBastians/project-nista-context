# Frontend Changelog

---

## 2026-04-09 — Feed Category Filter (Option A: Filter Icon + Bottom Sheet)

### Summary
Added category filtering to the feed page alongside the existing occasion filter. The filter icon sits pinned at the right of the occasion pill row and opens a bottom sheet with category chips.

### Files Changed

**`src/lib/store.tsx`**
- Added `categoryFilter: "all" | string` to `AppState` interface
- Added `categoryFilter: "all"` to `INITIAL_STATE`
- Added `SET_CATEGORY_FILTER` to the `Action` union type
- Added `case "SET_CATEGORY_FILTER"` to the reducer
- Added `categoryFilter: raw.categoryFilter ?? "all"` to the `HYDRATE` migration (backwards-compatible with older persisted state)

**`src/app/(app)/feed/page.tsx`**
- Imported `CATEGORIES` from `@/lib/mock-data`
- Imported `SlidersHorizontal` from `lucide-react`
- Read `categoryFilter` from store state; added `filterSheetOpen` boolean local state
- Added `setCategoryFilter()` function dispatching `SET_CATEGORY_FILTER`
- Restructured the sticky filter row: occasion pills in a `flex-1` scrollable div, `SlidersHorizontal` icon button pinned at the right behind a divider — renders filled/inverted when a category is active
- Extended `sortedProducts` useMemo to filter by `categoryFilter` in addition to `occasionFilter`; added `categoryFilter` to the dependency array
- Updated empty state: shows "No items match your filters / Try adjusting your occasion or category filters." when either filter is active; retains dismissal copy otherwise
- Added category bottom sheet: fixed backdrop (tap to close) + sheet with drag handle, "Category" header, X button, and chips for "All Categories" + the 8 `CATEGORIES` entries; selecting a chip applies the filter and auto-closes the sheet

### Behaviour
- Filter icon turns solid black when a category is active, providing clear visual feedback
- Occasion and category filters are ANDed together — both can be active simultaneously
- Filter state persists to `localStorage` via the existing store hydration mechanism
- Category filter resets to "all" only if the user explicitly selects "All Categories" in the sheet
