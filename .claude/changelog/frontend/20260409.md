## Bottom Nav + Header Restructure

**Request**
Remove Profile from the bottom nav to free a slot for a future 5th feature. Replace with a 4-tab bar (Feed, Search, Wishlist, Bag) with Bag anchored at the far right for conversion prominence. Profile access moves to a persistent icon in the page header. The header gets a centered STROBE logo mark (no text) and notification bell on the right — a layout consistent with premium fashion app conventions (SSENSE, Net-a-Porter).

**Solution**
Created a shared `AppHeader` component with three zones: profile icon (left, links to `/profile`), STROBE blue-square mark absolutely centred, notification bell (right). Applied to Feed, Wishlist, and Orders pages. Profile page retains its own header but gains a back button since it is no longer a bottom nav tab. Bottom nav trimmed to 4 tabs and `/notifications` added to hidden routes.

## Feed Filter Row — Hide on Scroll-down, Reveal on Scroll-up

**Request**
The filter row (occasion selector + category slider icon) stays visible while scrolling, consuming vertical space. Requested to hide during scroll like Instagram's secondary chrome behaviour.

**Solution**
Added scroll-direction detection using a `requestAnimationFrame`-debounced scroll listener (coexists with the existing scroll-save handler). A 5px threshold prevents jitter. State: `filterHidden` (false when at top y<10, true when scrolling down, false when scrolling up). The filter row outer div remains `sticky top-[53px]` for positioning; an inner wrapper uses `overflow-hidden` + `max-h-[60px]`/`max-h-0` transition (200ms ease-out) to collapse the height — this avoids the transform/gap issue that arises with sticky + translateY. When collapsed, the sticky element occupies 0 height and the feed grid fills the space below the AppHeader.

**Files impacted**
- `src/app/(app)/feed/page.tsx` — Added `filterHidden` state, `lastScrollY` ref, scroll-direction `useEffect`, restructured filter row JSX with collapsing wrapper

**Test**
- `npm run build` passes with 0 errors
- Scroll down: filter collapses smoothly (200ms), feed content shifts up
- Scroll up (any amount, >5px): filter reveals immediately
- At y<10: filter always visible regardless of direction

## Occasion Filter — Scrollable Pills → Single Selector Button

**Request**
On a 375px viewport the 5 occasion pills overflowed horizontally and the scroll affordance was invisible. Replace with a single tap-to-open selector button.

**Solution**
Replaced the scrollable pill row with a single rounded-full button showing the active occasion label ("All occasions" when unfiltered, label when filtered). Tapping opens a bottom sheet matching the existing category filter sheet pattern. Active (non-all) state uses `bg-ink text-canvas` to match old pill active styling.

**Files impacted**
- `src/app/(app)/feed/page.tsx` — Added `occasionSheetOpen` state; replaced pills with selector button; added occasion bottom sheet above category sheet; added `ChevronDown` import

## Drawer + Layout Safe Area Fixes (BUG-005, BUG-006)

**BUG-005** — Layout `pb-20` did not account for `env(safe-area-inset-bottom)`. Content was clipped behind the bottom nav on notched iPhones. Fixed: `pb-[calc(5rem+env(safe-area-inset-bottom))]` in `src/app/(app)/layout.tsx`.

**BUG-006** — Bottom drawer used `inset-x-0` (full viewport width). On laptop the drawer spanned edge-to-edge while app content was constrained to `max-w-lg`. Fixed: replaced with `left-[max(0px,calc(50vw-16rem))] right-[max(0px,calc(50vw-16rem))]` in `src/components/ui/drawer.tsx`.
