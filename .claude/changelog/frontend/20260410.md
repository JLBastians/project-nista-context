## Interactive Gesture Tutorial Overlay

**Request** — Replace the static "How to interact" gesture tooltip on the feed page (text-only, auto-dismisses after 8s) with an interactive tutorial that teaches gestures by having the user perform them on a real product card in their feed.

**Solution** — Built a 4-step coach-mark tutorial overlay (`GestureTutorial`) that renders fixed over the real feed on first visit after onboarding. Each step spotlights the first product card, shows an instruction, and waits for the user to perform the gesture before advancing. A hand-hint animation appears after 3 seconds of inactivity. Gestures are real actions (wishlist save, like, quick view), not sandbox interactions.

**Step sequence:** Swipe right → Swipe left → Double-tap → Long-press

**Key design decisions:**
- Tutorial overlays the real feed (not a fake card) — product card at index 0 is the tutorial target
- `onDismiss` is suppressed for the tutorial card during swipe-left step so the product stays in the grid
- Swipe-left gesture is detected and acknowledged without actually dismissing the product
- Body scroll locked during tutorial via `document.body.style.overflow = "hidden"`
- 4-panel touch-blocker technique: transparent `pointer-events: auto` divs around the spotlight card block interaction with non-tutorial cards, while the card itself remains fully touchable
- Step tracking lifted to feed page (`tutorialStep` state) so ProductCard receives the current expected gesture via `tutorialMode` prop — ensures correct gesture detection and side-effect handling per step
- `gestureTutorialComplete` replaces `gestureTooltipShown` in store with HYDRATE migration for existing users

**Files impacted:**
- `Frontend/src/lib/store.tsx` — Renamed `gestureTooltipShown` → `gestureTutorialComplete`, added `COMPLETE_GESTURE_TUTORIAL` action, kept `DISMISS_GESTURE_TOOLTIP` as backward-compat alias, updated HYDRATE migration
- `Frontend/src/components/product-card.tsx` — Exported `TutorialStep` type, added `tutorialMode` and `onTutorialGesture` optional props, wired all 4 gesture handlers to call `onTutorialGesture` when in tutorial mode; swipe-left suppresses dismiss during tutorial
- `Frontend/src/components/gesture-tutorial.tsx` — NEW: self-contained tutorial overlay with spotlight, coach mark, hand-hint SVG, confirmation chip, skip button, step progress dots, and reduced-motion support
- `Frontend/src/app/globals.css` — Added 5 new keyframes (`coach-mark-pulse`, `hand-swipe-right`, `hand-swipe-left`, `hand-double-tap`, `hand-longpress`) + utility classes + reduced-motion suppression + `animate-fade-out`
- `Frontend/src/app/(app)/feed/page.tsx` — Removed old gesture tooltip state/effect/JSX, added `tutorialCardRef`, `tutorialStep` state, `gestureHandlerRef`, wired `GestureTutorial` component with step-advance callback pattern

**Test:**

## Long-press: Moved from Pointer Events to Touch Events

**Request** — Long-press was not triggering the quick-view drawer on Android Chrome even after context menu suppression and 10px movement threshold on pointer events.

**Solution** — Root cause: mobile Chrome fires `pointerleave`/`pointercancel` during a stationary hold as the browser takes over gesture detection, cancelling the timer before 500ms. Fixed by removing pointer event handlers entirely and moving long-press detection to touch events: `handleTouchStart` starts a 500ms timer; `handleTouchMove` cancels if >10px movement; `handleTouchEnd` cancels timer before swipe/tap logic runs. Removed `handlePointerDown/Move/Up` functions and `onPointerDown/Move/Up/Leave` from JSX; added `onTouchMove={handleTouchMove}`.

**Files impacted:**
- `Frontend/src/components/product-card.tsx` — pointer handlers removed, touch-based long-press, `onTouchMove` added to JSX, `longPressTimer` cancelled at start of `handleTouchEnd`

**Test:** TypeScript: 0 errors. Long-press reliably opens quick-view drawer on Android Chrome without false cancellations.
