# Frontend Changelog — 2026-04-11

---

## Onboarding Gate Enforced at Route Level

**Request** — Ensure the onboarding quiz is triggered immediately after a user signs into the app, not just after registration.

**Solution** — Two changes: (1) Added a second `useEffect` to `AuthGuard` that redirects authenticated users with `onboardingComplete: false` to `/onboarding`. A `pathname !== "/onboarding"` check prevents an infinite redirect loop since the onboarding route itself also uses `AuthGuard`. (2) Fixed the `LOGIN` reducer case to preserve `onboardingComplete` from existing session state (`state.user?.onboardingComplete ?? false`), so returning users who already completed onboarding aren't sent back to the quiz every time they log in.

**Files impacted:**
- `Frontend/src/components/auth-guard.tsx` — added `usePathname` import and onboarding redirect effect
- `Frontend/src/lib/store.tsx` — `LOGIN` case now preserves `onboardingComplete` from existing state

**Test:** TypeScript: 0 errors. New user registers → onboarding. User who never completed onboarding logs in → redirected to onboarding. User who completed onboarding logs out and back in → lands on feed.
