## Feature 1: Animated Like/Save Feedback

**Files Modified:**
- `Frontend/src/app/globals.css`
  - Added keyframes: `heart-burst` (scale bounce 300ms), `glow-red` (box-shadow pulse 400ms), `glow-blue` (box-shadow pulse 400ms), `fade-out` (counterpart to fade-in)
  - Added utility classes: `.animate-heart-burst`, `.animate-glow-red`, `.animate-glow-blue`, `.animate-slide-right`
  - Updated `prefers-reduced-motion` block to disable all 4 new animations

- `Frontend/src/components/product-card.tsx`
  - Added `useState` import
  - Added state: `likeAnimating`, `bookmarkAnimating`, `quickViewOpen`, `preventNavigation` ref
  - Modified `handleLike()`: triggers animation + vibrate on positive action only
  - Modified `handleWishlist()`: triggers animation + vibrate when adding (not removing)
  - Updated heart button: conditionally apply `animate-heart-burst animate-glow-red`
  - Updated bookmark button: conditionally apply `animate-heart-burst animate-glow-blue`
  - Increased touch targets: heart/bookmark buttons from `h-8 w-8` to `h-11 w-11` (44×44px WCAG standard)
  - Added `cursor-pointer` class to Link wrapper

**Behaviour:**
- Tap heart on non-liked product → scale bounce 1→1.35→1 (300ms) + red glow + haptic
- Tap bookmark on non-saved product → scale bounce + blue glow + haptic
- Un-liking/removing does NOT animate
- Animations respect `prefers-reduced-motion`

## Feature 6: Quick-View Drawer

**Files Created:**
- `Frontend/src/components/quick-view-drawer.tsx` (new file, 165 lines)
  - Client component using vaul `Drawer` (already installed)
  - Props: `product: Product`, `open: boolean`, `onOpenChange: (open: boolean) => void`
  - Layout: thumbnail + product info (brand, name, price), size selector, Add to Bag CTA, wishlist toggle, View Details link
  - Uses `useBag()` for add-to-bag, `useWishlist()` for wishlist toggle
  - Shake animation on Add to Bag if no size selected
  - Toast confirmation on successful add with "View Bag" action
  - Auto-closes drawer and clears selectedSize on successful add

**Files Modified:**
- `Frontend/src/components/product-card.tsx`
  - Import `QuickViewDrawer`, `Plus` icon
  - Added `useState` for `quickViewOpen` (already added above, reused)
  - Changed long-press behaviour: 300ms timer now opens drawer instead of dismissing
  - Added `+` button as third action button in the card's top-right stack
  - `handleQuickView()` opens the drawer
  - `preventNavigation` ref prevents link navigation when drawer opens on long-press
  - Click handler on Link checks `preventNavigation` to block navigation if set
  - Changed return to fragment `<>` wrapping both Link and QuickViewDrawer
  - Swipe-left dismiss unchanged

**Behaviour:**
- Long-press (300ms) on card opens quick-view drawer (does NOT navigate or dismiss)
- Tap `+` button on card opens same drawer
- Select size + Add to Bag → adds to bag, shows toast with "View Bag" action, closes drawer
- No size + Add to Bag → shake animation + error toast
- Short tap on card → still navigates to product detail
- Swipe-left → still dismisses with undo toast
- Drawer dismissible by dragging down or clicking X

## Feature 7: Smooth Page Transitions

**Files Modified:**
- `Frontend/next.config.mjs`
  - Added `experimental: { viewTransition: true }` to enable Next.js View Transitions API
  - Build log confirms flag enabled: `✓ viewTransition`

- `Frontend/src/app/globals.css`
  - Added keyframe `slide-from-right`: opacity + translateX (used as fallback)
  - Added `::view-transition-old(root)` rule: fade-out 150ms snap easing
  - Added `::view-transition-new(root)` rule: fade-in 150ms snap easing
  - Added `::view-transition-group(*)` rule: 250ms spring easing for shared-element transitions
  - Updated `prefers-reduced-motion` to disable all view transition animations

- `Frontend/src/components/product-card.tsx`
  - Added `style={{ viewTransitionName: \`product-image-${product.id}\` }}` to card image
  - Imported `Plus` icon (already listed above)

- `Frontend/src/app/(app)/product/[id]/page.tsx`
  - Added `style={{ viewTransitionName: \`product-image-${product.id}\` }}` to hero image

**Behaviour:**
- Feed → product detail: product image morphs from card position to hero position (View Transitions API)
- Product detail → feed: reverse morph
- Unsupported browsers: graceful fallback to fade-in-up
- With `prefers-reduced-motion: reduce`: no animations play
