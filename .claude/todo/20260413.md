## Phase 4: Verify tokenless Shopify access

Run this Python snippet to test each brand (use Python instead of curl — PowerShell's `curl` is `Invoke-WebRequest` and behaves differently):

```bash
python -c "
import requests, json
stores = [
    ('Gymshark',     'gymshark.myshopify.com'),
    ('Allbirds',     'allbirds.myshopify.com'),
    ('Reformation',  'thereformation.myshopify.com'),
]
query = '{\"query\": \"{ products(first: 1) { edges { node { id title } } } }\"}'
for name, domain in stores:
    url = f'https://{domain}/api/2026-04/graphql.json'
    try:
        r = requests.post(url, data=query, headers={'Content-Type': 'application/json'}, timeout=15)
        data = r.json()
        if 'data' in data:
            title = data['data']['products']['edges'][0]['node']['title']
            print(f'  OK  {name}: \"{title}\"')
        else:
            print(f'  FAIL {name}: {data}')
    except Exception as e:
        print(f'  ERROR {name}: {e}')
"
```

**What the results mean:**
- `OK  Gymshark: "Power Shorts"` → accessible, safe to enable
- `FAIL` with `Not Found` error → domain is wrong
- `FAIL` with HTML content → store has disabled public access, skip this brand

**Enable only the stores that returned OK.** Run in psql or any PostgreSQL client:

```sql
UPDATE brands SET is_active = TRUE WHERE name = 'Gymshark';
-- Repeat for each brand that passed
```

Or via command line:

```bash
psql postgresql://postgres:YOUR_PASSWORD@localhost:5432/strobe_dev -c "UPDATE brands SET is_active = TRUE WHERE name = 'Gymshark';"
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `sqlalchemy error: Expected string or URL object, got None` | Missing or empty `DATABASE_URL` in `.env` | Check `.env` exists and `DATABASE_URL` is set |
| Migration step errors on `brands` FK | Migration ran partially before | Re-run — idempotency checks skip already-applied steps |
| Store verification returns HTML | That store has disabled public API access | Don't enable that brand; try a different one |
| `Brand 'Gymshark' not found in brands table` | `--seed-brands` not run, or wrong DB | Run `--seed-brands`, confirm `DATABASE_URL` is correct |
| Dry run shows data but live ingest shows 0 inserted | `is_active = FALSE` on the brand | Run `UPDATE brands SET is_active = TRUE WHERE name = 'Gymshark';` |
| E2E Test 4 skipped | FastAPI not running | Start `uvicorn api.main:app --reload` in a second terminal |
