Electric Blue Usage Rule
Electric Blue (#0057FF) is permitted only on: new-item badges, focus rings, primary CTA buttons when paired with a black alternative, and the active state of navigation items. It must never be used as a background fill on large surfaces.

13 · Guidelines

Motion.

Motion is functional, never decorative. Transitions communicate state change. Duration and easing follow these tokens.

Instant
Immediate feedback. No visible delay. Used for toggles and checkbox state changes.
Hover me
duration: 0ms
Micro
Hover states, button press, icon swaps. Subtle and fast — barely perceptible.
Hover me
duration: 150ms · ease-out
Short
Card hover elevation, dropdown open, tooltip appear. Snappy and intentional.
Hover me
duration: 200ms · ease-out
Medium
Page transitions, modal entry, drawer slide. Fluid and visible but never slow.
Hover me
duration: 300ms · ease-in-out
Long
Hero reveals, splash screens, onboarding. Used only for major entry moments.
Hover me
duration: 500ms · ease-in-out
Spring
Sheet drag release, image zoom. Natural physics — only on touch or gesture events.
Gesture only
spring · stiffness: 300 · damping: 30
Motion Principles
· Reduce — always respect prefers-reduced-motion
· Purposeful — every transition must explain a state change
· Fast in, slow out — UI elements arrive quickly and leave naturally
· No loops — avoid looping animations except loading states
· Consistent — matching interactions use matching durations

15 · Guidelines

Never Change.

These elements form the immutable core of the STROBE identity. They must never be altered, overridden, or reinterpreted — in any context, on any surface.

✕
The Wordmark LetterformsNever alter, distort, skew, stretch, or restyle the STROBE wordmark. Only approved SVG assets may be used.
✕
The Logo Diamond IconThe geometric diamond mark cannot be redrawn, recoloured outside approved variants, or combined with other icons or shapes.
✕
Electric Blue as a FillElectric Blue (#0057FF) must never be used as a large background, gradient base, or fill on any surface larger than a badge or focus ring.
✕
The Primary TypefaceSpace Grotesk is the only permitted typeface. No substitute fonts, no decorative display fonts, no system fonts in branded contexts.
✕
Gendered LanguageNever use "men's", "women's", "his", "hers", or any gendered labels in product categorisation, navigation, or UI copy.
✕
Icon Fill StylesIcons must always be outline / stroke style. Never use filled icon variants — they break visual consistency with the STROBE language.
✕
Off-Brand Colour IntroductionsNo new brand colours may be introduced without a full design system review. The palette is closed.
✕
Logo on Busy BackgroundsThe STROBE logo may only appear on solid Canvas, Ink, or Electric backgrounds. Never on photography, patterns, or gradients.
✕
Spacing Outside the ScaleNever use arbitrary pixel values not in the base-4 spacing system. All spacing must reference a design token.
✕
Motion on Static ContentNever add looping or decorative animations to product images, text, or UI elements that are not communicating a state change.
✕
Reducing Size RepresentationThe platform must always display the full available size range. Never suppress or deprioritise extended sizes in filters or display.
✕
App Icon Redesign Without ApprovalThe app icon shape, geometry, and approved colour variants are locked. Never introduce a new icon variant without a formal brand sign-off process.
Who this applies to
These constraints apply to every team, partner, contractor, and third-party working with the STROBE brand — without exception. Any proposed change to items on this list requires written approval from the STROBE brand lead.
